1. OVERVIEW PROGRAM
================================================================================

Produktivia adalah aplikasi Flutter yang dirancang untuk manajemen produktivitas.
Aplikasi ini menggunakan sistem autentikasi dummy (in-memory) untuk pengembangan
dan testing. Aplikasi memiliki beberapa screen yang terhubung dengan navigasi
yang mulus menggunakan custom transitions.

FITUR UTAMA:
- Sistem autentikasi (register dan login)
- Splash screen dengan animasi
- Onboarding/Register screen
- Login screen
- Home screen
- OTP verification screen (tersedia tapi tidak digunakan dalam flow dummy)

TEKNOLOGI:
- Flutter SDK 3.9.2+
- Firebase (opsional, untuk production)
- Google Fonts untuk typography
- Material Design 3

4. PENJELASAN FILE UTAMA
================================================================================

4.1. main.dart
---------------
File ini adalah entry point aplikasi. Fungsi main() melakukan:
1. Inisialisasi WidgetsFlutterBinding
2. Inisialisasi Firebase (opsional, dengan error handling)
3. Menjalankan aplikasi dengan MyApp widget

MyApp Widget:
- Mengatur MaterialApp dengan theme
- Menggunakan Google Fonts (Inter) untuk semua text
- Menetapkan SplashScreen0 sebagai home screen awal
- Material Design 3 enabled

4.2. models/user_model.dart
---------------------------
Model data untuk user dengan struktur:
- id: String - Unique identifier user
- email: String - Email user
- displayName: String - Nama tampilan user
- phoneNumber: String? - Nomor telepon (opsional)
- createdAt: DateTime - Waktu pembuatan akun

Method yang tersedia:
- fromFirestore(): Convert dari Firestore document
- toFirestore(): Convert ke Firestore document
- fromJson(): Convert dari JSON
- toJson(): Convert ke JSON

4.3. services/dummy_auth_service.dart
-------------------------------------
Service autentikasi dummy yang menyimpan data di memory (tidak persisten).

FITUR:
- signUp(): Registrasi user baru
  * Validasi email unik
  * Validasi password minimal 6 karakter
  * Simpan user di memory Map
  * Set sebagai current user

- signIn(): Login user
  * Cek email terdaftar
  * Validasi password
  * Set sebagai current user

- signOut(): Logout user
  * Clear current user

- currentUser: Getter untuk user yang sedang login
- authStateChanges: Stream untuk perubahan auth state
- clearAllUsers(): Method untuk testing (clear semua data)

PENTING:
- Data hilang saat aplikasi ditutup (in-memory storage)
- Password disimpan plain text (HANYA untuk development!)
- Tidak ada enkripsi atau hashing

4.4. services/auth_service.dart
--------------------------------
Service autentikasi Firebase (backup, tidak digunakan saat ini).
Berisi implementasi lengkap untuk Firebase Auth dan Firestore.
Dapat digunakan untuk production dengan mengganti import di screens.

5. ALUR APLIKASI (FLOW)
================================================================================

ALUR STARTUP:
1. main.dart → MyApp → SplashScreen0
   └─> Tampilkan logo Produktivia di tengah (1.5 detik)

2. SplashScreen0 → SplashScreen (fade transition)
   └─> Animasi logo bergerak dari center ke center-left (0.8 detik)
   └─> Delay tambahan 0.5 detik

3. SplashScreen → SplashScreen2 (no transition)
   └─> Logo di posisi kiri, text "PRODUKTIVIA" fade in
   └─> Cek apakah user sudah login:
       ├─> Jika sudah login → HomeScreen
       └─> Jika belum login → LoginScreen

ALUR REGISTRASI:
1. LoginScreen → Klik "Create an Account" → OnboardingScreen
2. OnboardingScreen:
   - Input: Display Name, Email, Password, Confirm Password
   - Validasi semua field
   - Klik "Create an Account"
   - DummyAuthService.signUp() dipanggil
   - Jika berhasil → HomeScreen
   - Jika error → Tampilkan SnackBar error

ALUR LOGIN:
1. LoginScreen:
   - Input: Email, Password
   - Validasi field
   - Klik "Log In"
   - DummyAuthService.signIn() dipanggil
   - Jika berhasil → HomeScreen
   - Jika error → Tampilkan SnackBar error

ALUR LOGOUT:
1. HomeScreen → Klik icon logout
2. DummyAuthService.signOut() dipanggil
3. Navigate ke LoginScreen

6. SISTEM AUTHENTICATION
================================================================================

SISTEM YANG DIGUNAKAN: Dummy Authentication (In-Memory)

CARA KERJA:
1. Data disimpan di static Map<String, Map<String, dynamic>> _users
2. Key Map adalah email (lowercase)
3. Value Map berisi:
   - 'user': UserModel object
   - 'password': String password (plain text)

REGISTRASI:
- Email diubah ke lowercase untuk konsistensi
- Password minimal 6 karakter
- Email harus unik (tidak boleh duplikat)
- User ID dibuat dari timestamp (millisecondsSinceEpoch)

LOGIN:
- Email diubah ke lowercase untuk matching
- Password dibandingkan secara langsung (plain text)
- Jika cocok, user diset sebagai currentUser

SESSION:
- currentUser disimpan di static variable
- Session hilang saat aplikasi ditutup
- Tidak ada token atau persistent storage

KEAMANAN:
⚠️ PERINGATAN: Sistem ini HANYA untuk development/testing!
- Password tidak di-hash
- Data tidak persisten
- Tidak ada enkripsi
- JANGAN digunakan di production!

UNTUK PRODUCTION:
Gunakan auth_service.dart yang sudah terintegrasi dengan Firebase Auth.

7. DETAIL SETIAP SCREEN
================================================================================

7.1. SplashScreen0 (splash_screen_0.dart)
-----------------------------------------
TIPE: StatefulWidget
DURASI: 1500ms (1.5 detik)

FUNGSI:
- Screen pertama yang muncul saat aplikasi dibuka
- Menampilkan logo Produktivia di tengah layar
- Background: Light purple (#DDA0DD)

ANIMASI:
- Tidak ada animasi, hanya static display

NAVIGASI:
- Setelah 1.5 detik → SplashScreen (dengan FadePageRoute)

KODE PENTING:
- initState() memanggil _navigateToNext()
- Future.delayed untuk delay 1500ms
- pushReplacement untuk mengganti screen

7.2. SplashScreen (splash_screen.dart)
---------------------------------------
TIPE: StatefulWidget dengan SingleTickerProviderStateMixin
DURASI: 800ms animasi + 500ms delay = 1300ms total

FUNGSI:
- Menampilkan logo dengan animasi perpindahan posisi
- Background: Light purple (#DDA0DD)

ANIMASI:
- Logo bergerak dari Alignment.center ke Alignment.centerLeft
- Menggunakan AnimationController dan Tween
- Curve: Curves.easeInOut
- Duration: 800ms

NAVIGASI:
- Setelah animasi selesai → SplashScreen2 (dengan NoTransitionRoute)

KODE PENTING:
- AnimationController untuk mengontrol animasi
- AnimatedBuilder untuk rebuild saat animasi berjalan
- NoTransitionRoute untuk transisi seamless

7.3. SplashScreen2 (splash_screen_2.dart)
-----------------------------------------
TIPE: StatefulWidget dengan SingleTickerProviderStateMixin
DURASI: 5ms delay + 400ms fade animation + 500ms delay = ~905ms

FUNGSI:
- Menampilkan logo di kiri dan text "PRODUKTIVIA" yang fade in
- Background: Light purple (#DDA0DD)
- Mengecek status login user

ANIMASI:
- Text "PRODUKTIVIA" fade in dari opacity 0 ke 1
- Duration: 400ms
- Curve: Curves.linear

LOGIKA AUTHENTICATION CHECK:
1. Buat instance DummyAuthService
2. Cek currentUser
3. Jika ada → Navigate ke HomeScreen
4. Jika tidak ada → Navigate ke LoginScreen

NAVIGASI:
- Jika logged in → HomeScreen
- Jika tidak logged in → LoginScreen

KODE PENTING:
- FadeTransition untuk animasi text
- DummyAuthService untuk cek login status
- pushReplacement untuk navigasi

9. MODEL DATA

9.1. UserModel (models/user_model.dart)
----------------------------------------
Model untuk representasi data user.

PROPERTIES:
- id: String - Unique identifier
- email: String - Email address
- displayName: String - Nama tampilan
- phoneNumber: String? - Nomor telepon (nullable)
- createdAt: DateTime - Waktu pembuatan

METHODS:
1. fromFirestore(Map<String, dynamic> data, String id)
   - Convert dari Firestore document
   - Handle Timestamp conversion
   - Return UserModel instance

2. toFirestore()
   - Convert ke format Firestore
   - Convert DateTime ke Timestamp
   - Return Map<String, dynamic>

3. fromJson(Map<String, dynamic> json)
   - Convert dari JSON
   - Handle DateTime parsing
   - Return UserModel instance

4. toJson()
   - Convert ke JSON format
   - Convert DateTime ke ISO8601 string
   - Return Map<String, dynamic>

PENGGUNAAN:
- Digunakan di DummyAuthService untuk menyimpan user data
- Dapat digunakan dengan Firebase Firestore jika diperlukan
